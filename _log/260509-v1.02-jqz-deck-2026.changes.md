## What changed

### CONTEXT.md (novo arquivo)

**Before:** não existia  
**After:** PRD completo em `deck/CONTEXT.md` — estrutura de slides, tokens de brand, dimensionamento, animações, convenções de imagem, build/deploy e estratégia de versionamento para GitHub  
**Why:** necessário para subir o projeto em repositório GitHub com contexto completo

### Responsive self-scaling

**Before:** slides fixos em 1280×720px sem adaptação ao viewport  
**After:** script JS envolve cada `.slide` num `.slide-wrap`, aplica `transform: scale(Math.min(vw/1280, 1))` com `transformOrigin: top center` e força `height = 720 * scale` no wrapper para compensar o colapso de layout. No `@media print`, `.slide-wrap` reseta para `height: auto`  
**Why:** deck precisa escalar proporcionalmente em telas menores sem quebrar o layout interno de nenhum slide

### Slide 17 · Vale Mãe Maria 13kV

**Before:** tentativa de grid de 3 fotos empilhadas que quebrou o project-body (sem altura explícita, as divs filhas não tinham referência de altura)  
**After:** layout original restaurado — uma imagem com `object-fit: cover`, `object-position: center center`  
**Why:** imagens portrait (317×423px) com object-fit:cover ficam bem num container landscape quando a imagem preenche o painel; o grid sem altura explícita havia colapsado o conteúdo e empurrado o footer para fora

### .project-body · min-height: 0

**Before:** `flex: 1; display: grid; ...` sem restrição de encolhimento  
**After:** `flex: 1; min-height: 0; display: grid; ...`  
**Why:** sem min-height:0 o flex item não encolhe abaixo do tamanho natural do conteúdo, empurrando o footer para fora dos 720px do slide

### .gitignore (novo arquivo)

**Before:** não existia  
**After:** exclui `_extracao/`, `_source/`, `dist/`, `*.pdf`, `.DS_Store`  
**Why:** pasta _extracao contém 76 imagens (~50MB) e _source tem o PPT original (57MB) — não devem ir para o repositório

### build-dist.sh

**Before:** referenciava apenas Netlify  
**After:** instruções de deploy para GitHub Pages e Netlify; comentários atualizados  
**Why:** repositório GitHub pode usar GitHub Pages como hosting direto
